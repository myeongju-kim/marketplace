# Domain Context Template

이 파일은 테스트 영향 분석 결과를 실제 테스트케이스로 연결하기 위한 입력입니다.
프로젝트별로 복사해서 값만 채우면 됩니다.

## 1. 핵심 도메인

- 도메인명:
- 한 줄 설명:
- 이번 변경과 가장 밀접한 업무 흐름:

## 2. 주요 액터

- 사용자:
- 관리자:
- 배치/스케줄러:
- 외부 연동 시스템:

## 3. 주요 엔티티와 상태

| 엔티티 | 주요 필드 | 상태/구분값 | 비고 |
|---|---|---|---|
| 예: 주문 | 주문번호, 결제상태 | CREATED, PAID, CANCELED | |

## 4. 정상 흐름

1. 사용자가 무엇을 한다.
2. 시스템이 무엇을 검증한다.
3. 어떤 상태가 저장/변경된다.
4. 어떤 후속 처리(이벤트, 알림, 정산 등)가 발생한다.

## 5. 예외 흐름

- 어떤 조건에서 실패해야 하는가
- 실패 시 어떤 상태가 유지/변경되어야 하는가
- 재시도 또는 롤백이 필요한가

## 6. 테스트에서 반드시 확인해야 할 규칙

- 금액/수량/한도 규칙:
- 권한 규칙:
- 중복 처리 규칙:
- 외부 연동 실패 처리 규칙:
- 데이터 정합성 규칙:

## 7. 변경사항과 연결할 힌트

- 변경된 클래스/메서드가 어떤 업무 단계와 연결되는가
- 이미 자주 깨졌던 회귀 포인트는 무엇인가
- 이번 변경에서 특히 위험한 분기는 무엇인가

## 8. 기존 테스트 자산

- 이미 존재하는 단위 테스트:
- 이미 존재하는 통합 테스트:
- 수동 QA 시나리오:
- 이번에 재사용 가능한 테스트 데이터/계정:
