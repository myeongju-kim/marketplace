# checklist

## 실행 전 확인

- [ ] PR 주소가 입력되었는가?
- [ ] 도메인 설명 파일이 있다면 경로가 유효한가?
- [ ] 현재 디렉터리가 git 저장소 내부인가?
- [ ] GitHub CLI 또는 git fetch가 가능한 환경인가?
- [ ] Java / Spring 프로젝트인가?

## 스크립트 실행 확인

- [ ] `scripts/tia-impact.sh <PR_URL> [DOMAIN_FILE]`을 실행했는가?
- [ ] `tia-report.md`가 생성되었는가?
- [ ] 변경 파일 목록이 정상적으로 출력되었는가?
- [ ] 영향 컴포넌트 후보가 비어 있지 않은가?
- [ ] 도메인 설명 파일이 있다면 경로가 리포트에 포함되었는가?

## 분석 확인

- [ ] 변경 클래스가 올바르게 추출되었는가?
- [ ] 변경 메서드 후보가 올바르게 추출되었는가?
- [ ] Call Graph 역방향 후보가 포함되었는가?
- [ ] DI 영향 후보가 포함되었는가?
- [ ] Repository / Mapper 후보가 포함되었는가?

## 테스트케이스 작성 확인

- [ ] 도메인 설명이 있다면 변경사항 매핑이 작성되었는가?
- [ ] 도메인 설명이 없다면 추정/가정이 작성되었는가?
- [ ] 직접 테스트가 포함되었는가?
- [ ] 상위 호출자 테스트가 포함되었는가?
- [ ] 주요 도메인 변경이면 Integration Test가 포함되었는가?
- [ ] 불필요한 테스트를 과하게 포함하지 않았는가?
- [ ] 각 테스트 선정 이유가 작성되었는가?
- [ ] 각 테스트케이스에 사전조건, 절차, 기대결과가 있는가?

## 최종 산출물 확인

- [ ] 변경 요약이 있는가?
- [ ] `tia-test-cases.md` 파일이 작성되었는가?
- [ ] 테스트케이스 표 또는 목록 형태로 정리되었는가?
- [ ] 영향 컴포넌트가 정리되었는가?
- [ ] 추천 테스트 또는 실행 대상 테스트가 있는가?
- [ ] 각 테스트의 이유와 도메인 리스크가 있는가?
- [ ] 도메인 설명이 없다면 저신뢰/추정 문구가 있는가?
- [ ] 한계 또는 누락 가능성이 명시되었는가?
