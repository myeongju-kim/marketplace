# TDD Workflow

## 1. Analyze
- 요구사항을 테스트 가능한 단위로 나눕니다.
- 정상, 경계, 실패 케이스를 분리합니다.
- 테스트 대상의 공개 인터페이스를 먼저 고정합니다.

## 2. Red
- 실패해야 하는 테스트를 먼저 작성합니다.
- 테스트 이름에 기대 동작을 드러냅니다.
- 실패 원인이 요구사항과 직접 연결되는지 확인합니다.

## 3. Green
- 테스트를 통과하는 최소 코드만 작성합니다.
- 불필요한 추상화나 일반화는 미룹니다.
- 새 테스트가 필요한 변경만 허용합니다.

## 4. Refactor
- 중복 제거, 이름 개선, 구조 정리를 수행합니다.
- 동작 변화 없이 내부 품질만 개선합니다.
- 각 리팩터링 단계 후 테스트를 다시 실행합니다.

## 5. Verify
- 관련 테스트 스위트를 다시 실행합니다.
- 요구사항이 테스트로 모두 표현됐는지 확인합니다.
- 누락된 예외 케이스가 있으면 Red 단계로 돌아갑니다.
